<task type="auto">
  <name>Task 1: Wrap main.js hooks with try-catch and surface recovery suggestions</name>
  <files>scripts/main.js, lang/en.json, lang/it.json</files>
  <action>
1. **Wrap `Hooks.once('init')` (line 762)** in a top-level try-catch. On catch, call `console.error` with module prefix AND `ui.notifications.error()` with localized message. Use `{ permanent: true }` only if the error has `recoverySuggestions`. The init hook registers settings first so ui.notifications should be available.

2. **Wrap the async IIFE in `Hooks.once('ready')` (line 806)** with a single outer try-catch around the entire IIFE body. The existing inner try-catches for Phase 1 (TVA cache) and Phase 2 (index build) remain -- the outer catch is a safety net for any uncaught errors that escape those blocks. On catch:
   - `console.error(\`${MODULE_ID} | Background initialization failed:\`, error)`
   - If `error.recoverySuggestions?.length > 0`: join suggestions and append to notification message, use `{ permanent: true }`
   - Otherwise: `ui.notifications.error(message, { permanent: false })`

3. **Wrap `Hooks.on('getSceneControlButtons')` (line 908)** body in try-catch. It is synchronous but should be wrapped for safety. On catch, log error and show non-permanent notification. Do NOT re-throw (would crash Foundry's hook chain).

4. **Update recovery suggestion surfacing** in the existing error dialog path (main.js around line 740 area in `processTokenReplacement`). When a caught error has `recoverySuggestions`, append them to the `ui.notifications.error()` message: `${error.message}. Try: ${error.recoverySuggestions.join('. ')}`.

5. **Add localization keys** to `lang/en.json` under `notifications`:
   - `"initFailed"`: `"Token Replacer FA: Initialization failed. {error}"`
   - `"backgroundFailed"`: `"Token Replacer FA: Background setup failed. {error}"`
   - `"workerFallback"`: `"Token Replacer FA: Background worker failed, using slower method."`
   Add matching Italian translations to `lang/it.json`.

**IMPORTANT pitfall avoidance:**
- Do NOT add `ui.notifications` calls inside service methods -- only at catch boundaries in main.js hooks and processTokenReplacement. Services throw; callers notify. (Avoids double notifications per Pitfall 2 in research.)
- Do NOT use `{ permanent: true }` for warnings or transient errors -- only for errors with recovery suggestions.
- IndexWorker.js bare throws are ACCEPTABLE -- do not touch that file.
