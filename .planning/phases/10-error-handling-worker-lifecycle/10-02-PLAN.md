   ```javascript

       game.i18n.localize('TOKEN_REPLACER_FA.notifications.workerFallback') ||
   ```

   Per codebase audit: SearchOrchestrator.searchLocalIndexWithWorker is called from searchLocalIndex which has its own try-catch. Add `ui.notifications.warn()` in the catch block of the calling method (searchLocalIndex) when falling back from worker to non-worker search. Same localization key.

3. **main.js -- Add beforeunload cleanup:** After the ready hook (after line 901 area), add:
   ```javascript
   // Clean up Worker and storage on page unload
   window.addEventListener('beforeunload', () => {
     try {
       indexService.terminate();
     } catch { /* ignore cleanup errors */ }
     try {
       searchOrchestrator?.terminate();
     } catch { /* ignore cleanup errors */ }
   });
   ```
   Per Pitfall 4 from research: wrap each cleanup in try-catch, do NOT await async operations (beforeunload is synchronous). Use optional chaining on searchOrchestrator since it may not be exported as a singleton.

   Check how indexService and searchOrchestrator are referenced in main.js. If they are imported singletons, use them directly. If they are accessed via tokenReplacerApp properties, use that path.
