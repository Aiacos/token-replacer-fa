Create `scripts/types/typedefs.js` with JSDoc `@typedef` blocks for all shared data structures. This file has NO imports and NO exports of runtime values — only type definitions. It must be side-effect free.

Include these typedefs (from research Pattern 4):

1. **CreatureInfo** — `tokenId` (string), `tokenName` (string), `actorName` (string), `creatureType` (string), `creatureSubtype` (string), `currentImage` (string)

3. **IndexedCache** — `version` (number), `timestamp` (number), `lastUpdate` (number), `categories` (Object with nested category/subcategory structure mapping to arrays of {path, name}), `allPaths` (Object mapping path to {name, category, subcategories}), `termIndex` (Object mapping term to string[])

4. **ModuleError** — `errorType` (string), `message` (string), `details` (string, optional), `recoverySuggestions` (string[], optional)

5. **TVACacheEntry** — `path` (string), `name` (string), `category` (string) — the normalized form of TVA cache items

6. **SearchResult** — `matches` (TokenMatch[]), `searchType` (string), `query` (string), `duration` (number, optional)

Export an empty object at the end so the file is a valid ES module that services can import for type references:
