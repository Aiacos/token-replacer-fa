  <behavior>
  Test organization (describe blocks):

  **IndexedDB path (isIndexedDBSupported = true, default with fake-indexeddb):**
  - openDatabase(): opens connection, reuses cached connection on second call
  - testConnection(): returns true, database info accessible
  - has(): returns true for existing key, false for missing key
  - save(): stores data and returns true; stores timestamp metadata (number, roughly current)
  - load(): returns saved data (round-trip); returns null for missing key
  - remove(): deletes data and returns true; has() returns false after remove
  - clear(): removes all records from IndexedDB; has() returns false for all cleared keys
  - close(): nullifies db and dbPromise

  **localStorage fallback (isIndexedDBSupported set to false):**
  - has(): returns true for existing localStorage key, false for missing
  - save(): stores to localStorage and returns true; JSON format includes data and timestamp
  - save(): returns false for data exceeding 4.5MB size limit
  - load(): returns saved data (round-trip via localStorage); returns null for missing key
  - remove(): removes key from localStorage
  - clear(): removes only 'token-replacer-fa'-prefixed keys, leaves other keys intact

  **Migration:**
  - needsMigration(): returns true when old localStorage key exists and new key is empty
  - needsMigration(): returns false when no old key exists
  - needsMigration(): returns false when new key already has data
  - migrateFromLocalStorage(): full round-trip -- seed localStorage with {data, timestamp}, call migrate, verify data in IndexedDB via load(), verify old localStorage key removed
  - migrateFromLocalStorage(): returns false when old key does not exist
  - migrateFromLocalStorage(): returns false when old key has invalid format (no data property)

  **Error scenarios:**
  - IndexedDB open failure falls through to localStorage for save/load/remove
  - Transaction abort on save does not silently lose data (falls through to localStorage)

  **Singleton:**
  - Exported storageService is an instance of StorageService
  - storageService.isIndexedDBSupported is true (fake-indexeddb loaded)
  </behavior>
  <action>
Create `tests/services/StorageService.test.js` with the describe blocks above. Follow these patterns from the research:

**Instance isolation:** Create a fresh `new StorageService()` in beforeEach per describe block. Call `service.close()` in afterEach. Use `indexedDB.deleteDatabase('token-replacer-fa')` in afterEach for IndexedDB test groups to prevent state leakage between describe blocks.

**localStorage cleanup:** Call `localStorage.clear()` in beforeEach/afterEach for the fallback and migration describe blocks.

**localStorage fallback testing:** Set `service.isIndexedDBSupported = false` after construction to force the localStorage code path. This is the pattern from the research -- do NOT mock indexedDB globally, just flip the flag.

**Error simulation:** Use `vi.spyOn(service, 'openDatabase').mockRejectedValue(new Error('DB open failed'))` to simulate IndexedDB failures and test fallthrough to localStorage.

**Transaction abort:** Open the database, spy on `db.transaction` to return a transaction that aborts after a microtask. Verify the save operation still succeeds via localStorage fallback.

**Timestamp verification:** After save(), open the database directly via `service.openDatabase()`, create a readonly transaction, get the raw record, and assert `record.timestamp` is a number between `before` and `after` Date.now() captures.

**Size limit test:** Set `isIndexedDBSupported = false`, create data that serializes to >4.5MB (e.g., `'x'.repeat(5 * 1024 * 1024)`), call save(), assert returns false.

**Selective clear test:** Set `isIndexedDBSupported = false`, seed localStorage with both `token-replacer-fa-*` keys and unrelated keys, call clear(), verify only module-prefixed keys are removed.

**Singleton tests:** Import `{ storageService }` from the module. Assert `instanceof StorageService`. Assert `storageService.isIndexedDBSupported === true`.

Do NOT test connection lifecycle events (onblocked, onversionchange, onclose) -- deferred to Phase 10 per user decision.
