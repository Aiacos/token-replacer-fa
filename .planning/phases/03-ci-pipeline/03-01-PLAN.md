Create the `.github/workflows/` directory and write `ci.yml` with the following specification:

**Workflow name:** `CI`

**Triggers (locked decisions):**
- `push` to branches: `[develop, main]`
- `pull_request` to branches: `[develop, main]`
- `workflow_dispatch` (manual trigger -- Claude's discretion, adds flexibility for debugging)

**Concurrency (Claude's discretion):**
- Group: `${{ github.workflow }}-${{ github.event.pull_request.number || github.ref }}`
- `cancel-in-progress: true` -- saves CI minutes on rapid pushes

**Job:** Single job named `quality-gate`, runs-on `ubuntu-latest`

**Steps (in this order -- lint first for fastest feedback):**
1. **Checkout** -- `actions/checkout@v6`
2. **Setup Node.js** -- `actions/setup-node@v6` with `node-version: '22'` and `cache: 'npm'` (explicit cache required because package.json lacks `packageManager` field)
3. **Install dependencies** -- `npm ci` with `id: install` (needed for step outcome references)
4. **Lint** -- `npm run lint` with `if: always() && steps.install.outcome == 'success'`
5. **Format check** -- `npm run format:check` with `if: always() && steps.install.outcome == 'success'`
6. **Test** -- `npm test` with `if: always() && steps.install.outcome == 'success'`
7. **Typecheck** -- `npm run typecheck` with `if: always() && steps.install.outcome == 'success'`

**Critical details:**
- Do NOT use `continue-on-error: true` on any quality step -- it masks failures from job status. Use `if: always()` instead per research findings.
- Use `npm ci` (not `npm install`) for clean, reproducible installs.
- The typecheck script already includes `|| true` in package.json -- do NOT add it again in the workflow step. Just run `npm run typecheck`.
- All quality steps guard on `steps.install.outcome == 'success'` so they skip (rather than produce misleading errors) if npm ci fails.
