<task type="auto">
  <name>Task 2: Create Vitest, ESLint, and TypeScript config files</name>
  <files>vitest.config.js, eslint.config.js, jsconfig.json</files>
  <action>
1. Create `vitest.config.js` (ES module):
   ```javascript
   import { defineConfig } from 'vitest/config';

export default defineConfig({
test: {
environment: 'jsdom',
globals: true,
include: ['tests/**/*.test.js'],
},
});

````
- `environment: 'jsdom'` — required for Foundry global mocking in Phase 2
- `globals: true` — makes describe/it/expect available without import
- `include` pattern uses `tests/` mirror structure per user decision

2. Create `eslint.config.js` (ESLint v10 flat config, ES module):
- Import `@eslint/js`, `globals`, and `eslint-config-prettier/flat`
- Define `foundryGlobals` object with all Foundry VTT globals as `'readonly'`: game, ui, canvas, Hooks, foundry, renderTemplate, loadTemplates, CONFIG, CONST, Dialog, FormApplication, Application, FilePicker, ChatMessage
- First config block for `scripts/**/*.js` EXCLUDING `scripts/workers/**/*.js`:
  - ecmaVersion: 2022, sourceType: 'module'
  - globals: spread `globals.browser` + spread `foundryGlobals`
  - rules: spread `js.configs.recommended.rules` + `'no-unused-vars': ['warn', { argsIgnorePattern: '^_' }]`
- Second config block specifically for `scripts/workers/**/*.js`:
  - ecmaVersion: 2022, sourceType: 'module'
  - globals: spread `globals.dedicatedWorker` (NOT globals.browser — Workers don't have window/document)
  - Same rules as main block
- Third entry: `eslintConfigPrettier` (MUST be last — disables formatting rules that conflict with Prettier)

CRITICAL: Worker files use `globals.dedicatedWorker`, not `globals.browser`. Using browser globals on Worker files causes false `no-undef` errors for `self` and `postMessage`.

IMPORTANT: ESLint 10 adds 3 new recommended rules (`no-unassigned-vars`, `no-useless-assignment`, `preserve-caught-error`) not in ESLint 9. If any of these produce errors on existing code, they MUST be overridden to `'warn'` or `'off'` in the rules section. The goal is zero lint errors, not zero warnings.

3. Create `jsconfig.json` for `tsc --noEmit` type checking:
```json
{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": true,
    "noEmit": true,
    "target": "ES2022",
    "module": "ES2022",
    "moduleResolution": "bundler",
    "lib": ["ES2022", "DOM", "DOM.Iterable", "WebWorker"],
    "types": ["@league-of-foundry-developers/foundry-vtt-types"],
    "strict": false,
    "skipLibCheck": true
  },
  "include": ["scripts/**/*.js"],
  "exclude": ["node_modules", "releases"]
}
````

- `strict: false` — fvtt-types v13 beta produces many false errors in strict mode
- `skipLibCheck: true` — avoids false failures from fvtt-types beta type definition gaps
- `lib` includes `WebWorker` for IndexWorker.js WorkerGlobalScope types
- Type errors from `tsc --noEmit` are expected and acceptable — the runner just needs to not crash
